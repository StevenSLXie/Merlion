## Codebase Patterns
- Use `src/runtime/usage.ts` as the single source of truth for shared cost-rate env parsing; runtime entrypoints should call `resolveUsageRatesFromEnv()` instead of duplicating `MERLION_COST_*` parsing.
- For E2E usage archives, derive cache-aware top-level totals with `summarizeUsageSamples()` plus `deriveUsageMetrics()` and keep per-turn cache-triage details in `prompt_observability` instead of inventing a second transcript format.
- When an archive-derived contract also drives gating, compute one `UsageDerivedMetrics` object and pass it to both the archive writer and the gate so review/debug output cannot drift from the persisted evidence.
- When validating archived cost-gate decisions, compare against the decision’s `observedMetric`: primary-gate outcomes should line up with `derived_totals.primary_metric_value`, while raw-token guardrails must line up with archive `totals.total_tokens`.
- Live cache validation should keep stable-prefix metrics as the hard requirement and treat provider `cached_tokens` metadata as opportunistic evidence, because some providers surface cache hits inconsistently across identical follow-up turns.
